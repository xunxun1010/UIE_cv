# UIE_cv
这是一个用来记录关于水下视觉增强的计算机视觉学习的仓库-基于论文WaveAlign: Wavelet-Domain Prior-Guided Color–Texture Alignment for Underwater Image Enhancement
#### 技术栈：Stable Diffusion Turbo (单步潜空间扩散模型) + LoRA 适配器 + 离散小波变换(DWT) + DCDD(低频色彩散度差异) + NDTSA(自然域纹理统计对齐)。

### 1.核心内容介绍
#### 1.1为什么用 DWT 而非 FFT？
**关键词触发网络**：`空间非均匀退化` → `傅里叶丢失位置信息` → `小波时频双局部化` → `为NDTSA铺路`  
水下退化是空间非均匀（近处轻、远处重；不同水质区域不同）退化，处理空间非均匀退化需要知道"退化发生在图像的哪个位置"，FFT将图像全局变换为频谱，空间维度被完全压缩
DWT能提供每个小波系数对应一个局部空间位置，频率+空间同时保留吗，NDTSA需要在空间局部区域做纹理统计对齐，DWT的分辨率保留是前提
#### 1.2DCDD模块的"可导性"——为什么用高斯软分配？
**关键词触发网络**：`硬分箱不可导` → `端到端反向传播` → `高斯核平滑` → `JS散度对称有界`  
我们的目标是用颜色分布先验约束扩散模型训练，颜色分布度量需要计算增强图像与参考图像的PDF距离，传统直方图用硬分箱：每个像素硬性落入一个bin → 阶跃函数，问题在阶跃函数梯度处处为0（除跳跃点无穷大）→ 无法反向传播，
解决高斯软分配每个像素对每个bin都贡献一个连续权重 → 处处可导。其中JS散度 对称、有界[0, log2]、不会像KL散度那样出现无穷大
**数学推导链**
1. 硬分箱：pixel → `bin_idx = floor(value / bin_width)` → 不可导（阶跃函数）
2. 高斯软分配：pixel value x → `w_k(x) = exp(-(x - c_k)² / (2σ²))` → 可导
3. 归一化得到PDF → JS散度：`JS(P||Q) = 0.5·KL(P||M) + 0.5·KL(Q||M)`，M = (P+Q)/2

#### 1.3 NDTSA模块——为什么用随机锚点+Stop Gradient？
**关键词触发网络**：`纹理样本差异大` → `固定先验导致趋同` → `随机自然锚点` → `Stop Gradient单向约束`  

**推演逻辑链**：
1. 低频（颜色）和高频（纹理）的本质区别？→ 水下色偏是系统性的（都偏蓝绿），纹理是内容依赖的（珊瑚≠鱼群≠沉船）
2. 所以低频可以用什么？→ 固定的域级颜色先验（DCDD）
3. 高频为什么不行？→ 如果也用固定先验，所有增强图像的纹理会被"拉向"同一个模式 → 纹理趋同
4. 解决方案？→ 用随机采样的自然图像作为样本级统计锚点 → 保持纹理多样性
5. 为什么加Stop Gradient？→ 确保信息流是单向的：自然参考只"告诉"增强分支该往哪走，自然图像自身不被反向更新"拉下水"



#### 1.4 主干网络——Stable Diffusion Turbo + LoRA

**关键词触发网络**：`DDPM多步慢` → `ADD蒸馏单步` → `潜空间压缩` → `LoRA低秩高效`

**推演逻辑链**：
1. 传统扩散模型（DDPM）的问题？→ 50-1000步迭代去噪 → 推理太慢，无法实用
2. SD Turbo怎么做到的？→ ADD（对抗扩散蒸馏）：用教师模型的多步去噪结果监督学生模型的单步去噪
3. 为什么在潜空间做？→ VAE将图像压缩8× → 计算量降低64×
4. LoRA的作用？→ 全量微调SD参数巨大 → LoRA只训练低秩矩阵ΔW=BA → 参数量~1%

**实验数据支撑**：timing_summary.csv → 512×512单张仅需0.53秒
